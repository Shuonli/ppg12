# Improved Iterative Bayesian Unfolding

## Abstract

This paper reviews the basic ideas behind a Bayesian unfolding published some years ago (D'Agostini 1995, NIM A 362 487) and improves their implementation. In particular, uncertainties are now treated at all levels by probability density functions and their propagation is performed by Monte Carlo integration. Thus, small numbers are better handled and the final uncertainty does not rely on the assumption of normality. Theoretical and practical issues concerning the iterative use of the algorithm are also discussed. The new program, implemented in the R language, is freely available.

## Key Results / Algorithm

### Core Iteration Formula

The algorithm applies Bayes' theorem to invert the smearing (response) matrix. Given:
- `lambda_ji = P(E_j | C_i)` -- response matrix element (probability that cause `C_i` produces effect `E_j`)
- `P(C_i | I)` -- prior probability for cause bin `i`
- `epsilon_i = sum_j lambda_ji` -- efficiency for cause `i`

The inverse probability (unfolding matrix) is:

```
theta_ij = P(C_i | E_j) = lambda_ji * P(C_i | I) / sum_k [lambda_jk * P(C_k | I)]
```

The unfolded spectrum is then:

```
x(C_i) = (1/epsilon_i) * sum_j [theta_ij * x(E_j)]
```

### Iteration Procedure

1. Start with an initial prior `P(C_i | I)` (e.g., flat or MC truth distribution)
2. Compute `theta_ij` from the prior and the response matrix
3. Unfold the observed spectrum to obtain `x(C_i)`
4. Use the resulting `x(C_i)` (normalized) as the new prior
5. Repeat until convergence

Regularization is achieved by **stopping iterations before convergence** -- early stopping acts as implicit regularization. Too many iterations recover the (wildly fluctuating) exact inverse. Typically 3-5 iterations suffice.

### Improved Uncertainty Propagation

The original algorithm used linear error propagation, which underestimates uncertainties for >1 iteration. The improved version:

1. Models the smearing matrix columns via **Dirichlet distributions**: `lambda_i ~ Dir(alpha_prior + x^MC_E | x(C_i)^MC)`
2. Models observed counts via **Gamma distributions**: `mu_j ~ Gamma(c_j + x(E_j), r_j + 1)`
3. Shares counts among cause bins via **Multinomial sampling**: `x_C | m_j ~ Mult(m_j, theta_j)`
4. Performs MC sampling to propagate all uncertainties simultaneously
5. Each sampling yields a complete unfolded spectrum; covariance is computed from the ensemble

This properly handles:
- Small number statistics
- Uncertainty in the response matrix from finite MC statistics
- Non-Gaussian posterior distributions
- Full covariance between unfolded bins

### Convergence Criteria

The paper discusses several strategies:
- Monitor chi2 between re-folded result and observed data (should decrease then plateau)
- Use the unfolded result's smoothness as a guide
- In practice, 3-5 iterations are typically sufficient; the result is often insensitive to the exact number

## Methodology

The algorithm discretizes the unfolding problem into cause bins `C_i` and effect bins `E_j`, treating each cause bin as an independent degree of freedom. The response matrix `Lambda` (dimensions `n_E+1 x n_C`, including an inefficiency/trash bin) encodes detector smearing from MC simulation.

The key "trick" is to apply Bayes' theorem to individual cause-effect pairs rather than to entire spectra, circumventing the intractable likelihood `P(x_E | x_C, Lambda)` which involves sums of multinomials without closed form.

The algorithm naturally handles:
- Multiple dimensions (cause-cells and effect-cells can be multi-dimensional)
- Background estimation (extra cause-cells for background sources)
- Inefficiency (via the "trash" bin `E_{n_E+1}`)
- Non-square matrices (n_E != n_C)

## Relevance to PPG12

- **Primary unfolding method**: PPG12 uses the iterative Bayesian method (via RooUnfoldBayes) to unfold the measured isolated photon pT spectrum for detector resolution and efficiency effects
- **Implementation**: RooUnfold wraps this algorithm; PPG12's `CalculatePhotonYield.C` calls RooUnfoldBayes with the response matrix built from photon MC (photon5/10/20 samples)
- **Iteration count**: PPG12 typically uses 4 iterations (the RooUnfold default, consistent with this paper's recommendation)
- **Response matrix**: Built from truth-reco pT pairs in signal MC, with `pT_bins_truth` extending beyond `pT_bins` (reco) for overflow handling
- **Error treatment**: The covariance matrix from unfolding propagates into the final cross-section systematic uncertainties
- **Prior**: RooUnfoldBayes uses the training truth distribution as initial prior (not flat), which should converge faster per this paper's discussion
