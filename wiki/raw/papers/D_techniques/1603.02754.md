# XGBoost: A Scalable Tree Boosting System

## Abstract

Tree boosting is a highly effective and widely used machine learning method. In this paper, we describe a scalable end-to-end tree boosting system called XGBoost, which is used widely by data scientists to achieve state-of-the-art results on many machine learning challenges. We propose a novel sparsity-aware algorithm for sparse data and weighted quantile sketch for approximate tree learning. More importantly, we provide insights on cache access patterns, data compression and sharding to build a scalable tree boosting system. By combining these insights, XGBoost scales beyond billions of examples using far fewer resources than existing systems.

## Key Results / Algorithm

### Regularized Learning Objective

For a dataset with n examples and m features, XGBoost uses K additive tree functions:

```
y_hat_i = phi(x_i) = sum_{k=1}^{K} f_k(x_i),   f_k in F
```

where `F = {f(x) = w_{q(x)}}` is the space of regression trees. `q(x)` maps an example to a leaf index, and `w` contains the leaf scores.

The regularized objective:

```
L(phi) = sum_i l(y_i, y_hat_i) + sum_k Omega(f_k)
```

where the regularization term is:

```
Omega(f) = gamma * T + (1/2) * lambda * sum_{j=1}^{T} w_j^2
```

Here `T` is the number of leaves, `gamma` penalizes tree complexity (number of leaves), and `lambda` is L2 regularization on leaf weights.

### Second-Order Approximation

At step t, the objective is approximated via Taylor expansion to second order:

```
L_tilde^(t) = sum_i [g_i * f_t(x_i) + (1/2) * h_i * f_t^2(x_i)] + Omega(f_t)
```

where `g_i = d l / d y_hat^(t-1)` and `h_i = d^2 l / d y_hat^(t-1)^2` are the first and second derivatives of the loss.

### Optimal Leaf Weights and Structure Score

For a fixed tree structure q, the optimal weight of leaf j:

```
w_j* = - (sum_{i in I_j} g_i) / (sum_{i in I_j} h_i + lambda)
```

The corresponding optimal objective value (structure score):

```
L_tilde^(t)(q) = -(1/2) * sum_{j=1}^{T} (sum_{i in I_j} g_i)^2 / (sum_{i in I_j} h_i + lambda) + gamma * T
```

### Split Finding -- Loss Reduction

The gain from splitting a node is:

```
L_split = (1/2) * [ G_L^2/(H_L + lambda) + G_R^2/(H_R + lambda) - (G_L + G_R)^2/(H_L + H_R + lambda) ] - gamma
```

where `G_L = sum_{i in I_L} g_i`, `H_L = sum_{i in I_L} h_i` are the gradient statistics for the left child, and similarly for the right child. The term `gamma` acts as a minimum gain threshold for splitting.

### Split Finding Algorithms

**Exact Greedy Algorithm (Algorithm 1):**
- For each feature, sort instances by feature value
- Linear scan to enumerate all possible splits
- Select the split with maximum gain
- Time complexity: `O(K * d * ||x||_0 * log n)` without blocks

**Approximate Algorithm (Algorithm 2):**
- Propose candidate split points via percentiles of feature distribution
- Map continuous features into buckets defined by candidates
- Aggregate gradient statistics within buckets
- Find best split among proposals
- Two variants: **global** (propose once at tree construction) and **local** (re-propose after each split)

### Shrinkage and Column Subsampling

Two anti-overfitting techniques beyond the regularized objective:
1. **Shrinkage (learning rate)**: Scale newly added tree weights by factor `eta` after each boosting step -- analogous to learning rate in SGD
2. **Column (feature) subsampling**: Randomly sample a fraction of features for each tree or each split (borrowed from Random Forest). Users report this prevents overfitting even more than row subsampling

### Sparsity-Aware Split Finding (Algorithm 3)

Handles missing values and sparse features in a unified way:
- Assign a default direction (left or right) for each split
- Only visit non-missing entries `I_k = {i in I | x_{ik} != missing}`
- Learn optimal default direction from data by enumerating both left-default and right-default
- Computation is linear in the number of non-missing entries
- 50x speedup vs naive on sparse data (one-hot encoded features)

### Weighted Quantile Sketch (Appendix A)

For the approximate algorithm, candidate split points must be found on weighted data (each instance weighted by `h_i`, the second derivative). The paper introduces a novel weighted quantile summary data structure that:
- Supports merge and prune operations with provable approximation guarantees
- Enables distributed and streaming quantile computation
- Finds the eps-approximate weighted quantiles needed for candidate split proposals

### System Design

**Column Block for Parallel Learning:**
- Data stored in compressed sparse column (CSC) format in memory blocks
- Each column pre-sorted by feature value (computed once, reused across iterations)
- Enables parallel split finding across features
- Reduces time from `O(K*d*||x||_0*log n)` to `O(K*d*||x||_0 + ||x||_0*log n)`

**Cache-Aware Access:**
- Gradient statistics accessed in feature-sorted order (non-contiguous memory)
- Internal buffer per thread for prefetching gradient statistics
- Mini-batch accumulation to reduce cache misses
- 2x speedup on large datasets (10M instances)

**Out-of-Core Computation:**
- Block compression (26-29% compression ratio) trades computation for disk I/O
- Block sharding across multiple disks with prefetching threads
- Enables processing of 1.7 billion examples on a single machine

### Benchmarks

| Dataset | n | m | Task |
|---------|---|----|------|
| Allstate | 10M | 4227 | Insurance claim classification |
| Higgs Boson | 10M | 28 | HEP event classification |
| Yahoo LTRC | 473K | 700 | Learning to rank |
| Criteo | 1.7B | 67 | CTR prediction |

XGBoost outperforms pGBRT, Spark MLLib, H2O, scikit-learn, and R GBM in both speed and accuracy. It is the only system supporting exact greedy + approximate (global/local) + out-of-core + sparsity-aware + parallel computation.

## Methodology

XGBoost implements gradient boosting with second-order Taylor approximation of the loss function, enabling efficient computation of optimal leaf weights and split scores in closed form. The regularization term (L1 on number of leaves via gamma, L2 on leaf weights via lambda) is integrated directly into the objective, not applied post-hoc. The system combines algorithmic innovations (sparsity awareness, weighted quantile sketch) with systems-level optimizations (column blocks, cache-aware access, out-of-core computation) for end-to-end scalability.

## Relevance to PPG12

- **Training algorithm**: PPG12's `main_training.py` uses XGBoost (`xgboost` Python package) as the classifier for photon identification BDT training
- **Hyperparameters** (from `config.yaml`):
  - `objective: binary:logistic` (binary classification, logistic loss)
  - `eval_metric: auc` (area under ROC curve)
  - `n_estimators: 750` (number of boosting rounds K)
  - `max_depth: 5` (maximum tree depth)
  - `learning_rate: 0.1` (shrinkage factor eta)
  - `subsample: 0.5` (row subsampling fraction)
  - `colsample_bytree: 0.6` (column subsampling per tree)
  - `reg_alpha: 5.0` (L1 regularization, not in the paper's formulation but added later)
  - `reg_lambda: 0.3` (L2 regularization lambda)
  - `grow_policy: lossguide` (leaf-wise growth, not level-wise)
  - `tree_method: hist` (histogram-based approximate algorithm)
- **ET-binned models**: PPG12 trains separate models for ET bins [6,10,15,20,25,35] GeV, each with the same XGBoost hyperparameters
- **25 features**: The feature set includes cluster shape variables (e-ratios, et1-4, weta/wphi), position (eta, vertexz), and energy (Et), applied to signal (photon MC) vs background (jet MC)
- **Model export**: Trained XGBoost models are exported to TMVA ROOT format for C++ inference via `TMVA::Experimental::RBDT` in `apply_BDT.C`
- **NPB score model**: Also trained with XGBoost, targeting non-physics background separation
