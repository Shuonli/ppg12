# TMVA -- Toolkit for Multivariate Data Analysis with ROOT

## Abstract

In high-energy physics, with the search for ever smaller signals in ever larger data sets, it has become essential to extract a maximum of the available information from the data. Multivariate classification methods based on machine learning techniques have become a fundamental ingredient to most analyses. Integrated into the analysis framework ROOT, TMVA is a toolkit which hosts a large variety of multivariate classification algorithms. Training, testing, performance evaluation and application of all available classifiers is carried out simultaneously via user-friendly interfaces. With version 4, TMVA has been extended to multivariate regression of a real-valued target vector.

## Key Results / Algorithm

### Supported Classifiers

TMVA provides a comprehensive set of MVA methods within a unified ROOT-integrated framework:

1. **Rectangular cut optimization** -- binary splits on individual variables
2. **Projective likelihood estimation** -- product of 1D PDFs
3. **Multi-dimensional likelihood** -- PDE range-search, PDE-Foam, k-NN
4. **Linear discriminants** -- H-Matrix, Fisher, LD, FDA
5. **Artificial neural networks** -- three MLP implementations
6. **Support vector machine (SVM)**
7. **Boosted/bagged decision trees (BDT)** -- the method most relevant to PPG12
8. **Predictive learning via rule ensembles (RuleFit)**
9. **Generic boost classifier** -- can boost any of the above

### Boosted Decision Trees (Section 8.12)

The BDT implementation in TMVA:
- **Tree building**: Binary splits on input variables, optimized by maximizing separation gain (Gini index, cross entropy, or misclassification)
- **Boosting algorithms**: AdaBoost (default), Gradient Boost, Bagging
- **AdaBoost**: Misclassified events receive higher weights; final score is weighted majority vote
- **Gradient Boost**: Each tree fits the residuals (negative gradient of the loss function) from previous trees; learning rate (`Shrinkage`) controls step size
- **Pruning**: Cost-complexity pruning to prevent overtraining
- **Key parameters**: `NTrees`, `MaxDepth`, `MinNodeSize`, `Shrinkage`, `BaggedSampleFraction`

### Training/Application Architecture

**Training phase** (Factory):
1. User provides signal and background ROOT trees or text files
2. Factory performs preprocessing (normalization, decorrelation, PCA, Gaussianization)
3. All booked methods are trained, tested, and evaluated simultaneously
4. Output: weight files (XML format) containing trained model parameters

**Application phase** (Reader):
1. Reader loads weight files
2. Input variables are set per event
3. `EvaluateMVA()` returns classifier response
4. Lightweight: can be embedded in any C++ analysis

**Standalone C++ classes** (Section 3.4):
- TMVA can generate standalone response classes that encode the model without requiring TMVA or ROOT at runtime
- This is the basis for the `TMVA::Experimental::RBDT` fast inference interface used in modern ROOT

### Data Preprocessing (Section 4)

Available transformations (can be chained):
- **Normalization**: Map variables to [0,1] or [-1,1]
- **Decorrelation**: Linear decorrelation using square root of covariance matrix
- **PCA**: Principal component decomposition
- **Gaussianization**: Transform variables to Gaussian shape

### Performance Evaluation

- **ROC curves**: Background rejection vs signal efficiency
- **Overtraining checks**: Compare training and test sample distributions
- **Variable ranking**: Method-specific importance measures
- **Benchmark tables**: Signal efficiency at fixed background rejection rates

### Model File Format

TMVA models are saved as XML weight files by default, but BDT models can also be stored in ROOT files. The `TMVA::Experimental::RBDT` class (introduced in newer ROOT versions) provides fast inference from ROOT-format BDT models, loading the tree ensemble into memory for efficient batch evaluation.

## Methodology

TMVA operates in a "factory" mode ensuring unbiased comparison: all methods see identical training/test splits and are evaluated by consistent metrics. The framework handles event weighting, negative weights, variable transformations, and cross-validation internally. The software is implemented in C++ within ROOT, with Python access via PyROOT.

## Relevance to PPG12

- **BDT inference engine**: PPG12's `apply_BDT.C` uses `TMVA::Experimental::RBDT` to load trained XGBoost models (exported from Python to TMVA ROOT format) and compute BDT scores for each cluster
- **Model format**: XGBoost models trained in Python (`main_training.py`) are converted to TMVA-compatible ROOT files (`binned_models/model_{name}_single_tmva.root`) using the `xgboost2tmva` export
- **RBDT.Compute()**: The fast inference call in `apply_BDT.C` evaluates the tree ensemble on feature vectors extracted from slimtree branches
- **11 model variants**: PPG12 trains multiple BDT variants (base, base_v0 through base_v3, base_E, etc.) with different feature subsets, all applied via TMVA::RBDT
- **NPB score model**: The NPB (non-physics background) model is also stored as a TMVA ROOT file and applied via the same RBDT interface
- **ET threshold**: Scores are only computed for clusters with `cluster_Et > 7 GeV`; below this threshold the score is set to -1
- **ET-binned models**: PPG12 uses ET-binned BDT models (edges from `bdt_et_bin_edges` in config); TMVA handles the per-event model selection
