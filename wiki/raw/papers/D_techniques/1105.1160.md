# Unfolding Algorithms and Tests Using RooUnfold

## Abstract

The RooUnfold package provides a common framework to evaluate and use different unfolding algorithms, side-by-side. It currently provides implementations or interfaces for the Iterative Bayes, Singular Value Decomposition, and TUnfold methods, as well as bin-by-bin and matrix inversion reference methods. Common tools provide covariance matrix evaluation and multi-dimensional unfolding. A test suite allows comparisons of the performance of the algorithms under different truth and measurement models.

## Key Results / Algorithm

### Supported Unfolding Methods

| Method | Class | Regularization | Parameter |
|--------|-------|---------------|-----------|
| Iterative Bayes | `RooUnfoldBayes` | Early stopping | Number of iterations (typically 4) |
| SVD | `RooUnfoldSvd` | Singular value cutoff | `k` (cutoff index) |
| TUnfold | `RooUnfoldTUnfold` | Tikhonov (0/1/2-order polynomial) | `tau` (auto via L-curve) |
| Bin-by-bin | `RooUnfoldBinByBin` | MC model | None (correction factors) |
| Matrix inversion | `RooUnfoldInvert` | SVD removal of small values | None |

### RooUnfoldBayes (Iterative Bayes)

Implements D'Agostini's method (arXiv:1010.0632). Key implementation choices:
- **Initial prior**: Uses training truth distribution (not flat), reaching optimum in fewer iterations
- **Default iterations**: 4 iterations usually sufficient
- **No smoothing** by default (can introduce bias)
- **Error propagation**: Full covariance matrix via Eq. (3)-(4) below

The unfolding matrix is:

```
M_ij = P(E_j|C_i) * n0(C_i) / [epsilon_i * sum_l P(E_j|C_l) * n0(C_l)]
```

The unfolded distribution:

```
n_hat(C_i) = sum_j M_ij * n(E_j)
```

### Corrected Error Propagation (Key Contribution)

The paper identifies and fixes a significant underestimation of uncertainties in D'Agostini's original method for >1 iteration. The error propagation matrix must include the dependence of the prior on the data:

```
d_n_hat(C_i)/d_n(E_j) = M_ij + sum_k M_ik * n(E_k) * [
    (1/n0(C_i)) * d_n0(C_i)/d_n(E_j) 
    - sum_l epsilon_l/n0(C_l) * d_n0(C_l)/d_n(E_j) * M_lk
]
```

The covariance matrix:

```
V(n_hat(C_k), n_hat(C_l)) = sum_{i,j} [d_n_hat(C_k)/d_n(E_i)] * V(n(E_i), n(E_j)) * [d_n_hat(C_l)/d_n(E_j)]
```

Without the second term (the iteration-dependent correction), errors are underestimated for >1 iteration. Fig. 5 of the paper demonstrates this: the corrected propagation agrees well with toy MC RMS.

### RooUnfoldSvd

Interface to ROOT's `TSVDUnfold` (Hocker & Kartvelishvili method):
- Response matrix inverted via SVD
- Regularization: smooth cutoff on small singular values: `s_i^2 -> s_i^2 / (s_i^2 + s_k^2)` where `k` defines the cutoff
- Normalization retained to minimize MC training sample uncertainties
- Error matrix includes contribution from finite MC statistics
- **Tuning required**: `k` must be optimized per distribution, binning, and statistics

### RooUnfoldTUnfold

Interface to ROOT's `TUnfold` (Schmitt):
- Matrix inversion with 0-, 1-, or 2-order polynomial regularization of neighboring bins
- RooUnfold handles packing of 2D/3D distributions and regularization matrix construction
- Automatic `tau` determination via L-curve scanning: `log10(chi2)` vs `log10(tau)`

### Error Calculation Methods

Three methods available (selectable via switch):
1. **Bin-by-bin errors** -- diagonal only, no correlations (fast but incomplete)
2. **Full covariance from error propagation** -- propagates measurement errors through unfolding
3. **MC toy covariance** -- generate toy datasets, unfold each, compute covariance from ensemble

Important caveat: the covariance matrix may be poorly conditioned. For chi2 calculations, the matrix inverse involves subtracting large similar numbers, amplifying numerical errors. The paper recommends using `TDecompSVD` for the inversion.

### Software Architecture

```
RooUnfoldResponse  -->  RooUnfold (base class)
  Fill(measured, truth)     |-- RooUnfoldBayes
  Miss(truth)               |-- RooUnfoldSvd
                            |-- RooUnfoldTUnfold
                            |-- RooUnfoldInvert
                            |-- RooUnfoldBinByBin
```

- **Response matrix**: Filled via `Fill(x_measured, x_true)` and `Miss(x_true)` (for inefficiency)
- **Training/unfolding separation**: Response object saved to disk via ROOT I/O streamers
- **Multi-dimensional**: Supports TH1D, TH2D, TH3D
- **Variable binning**: Different truth and measured bin widths supported
- **Overflow handling**: Option to include or exclude under/overflow bins

## Methodology

The package uses a simple object-oriented C++ design within ROOT. The response matrix is an independent object that can be trained in a separate program, serialized, and loaded for unfolding. All algorithms inherit from a common base class, ensuring consistent interfaces for error calculation, histogram handling, and result extraction.

Test suite uses deliberately mismatched training and test distributions (single Gaussian for training, double Breit-Wigner for testing) to stress-test unfolding performance. This challenges the methods with significant model dependence.

## Relevance to PPG12

- **Direct dependency**: PPG12 uses `RooUnfoldBayes` from this package for unfolding the photon pT spectrum
- **Response matrix filling**: PPG12 fills the response matrix from signal MC (photon5/10/20) truth-reco pairs in `CalculatePhotonYield.C`
- **Miss() handling**: Events passing truth cuts but failing reco cuts are registered via `Miss()` to account for efficiency
- **pT_bins_truth extension**: PPG12 uses extended truth bins (wider range than reco) to handle overflow, consistent with RooUnfold's overflow support
- **4 iterations**: PPG12 uses the recommended 4 iterations for Bayesian unfolding
- **Covariance**: The unfolded covariance matrix propagates into the cross-section systematic error budget
- **Alternative methods**: RooUnfoldSvd could serve as a cross-check (systematic variation) of the default Bayesian method
