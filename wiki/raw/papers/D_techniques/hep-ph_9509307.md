# SVD Approach to Data Unfolding

## Abstract

Distributions measured in high energy physics experiments are usually distorted and/or transformed by various detector effects. A regularization method for unfolding these distributions is re-formulated in terms of the Singular Value Decomposition (SVD) of the response matrix. A relatively simple, yet quite efficient unfolding procedure is explained in detail. The concise linear algorithm results in a straightforward implementation with full error propagation, including the complete covariance matrix and its inverse. Several improvements upon widely used procedures are proposed, and recommendations are given how to simplify the task by the proper choice of the matrix. Ways of determining the optimal value of the regularization parameter are suggested and discussed, and several examples illustrating the use of the method are presented.

## Key Results / Algorithm

### Problem Formulation

The measured distribution `b` (n_b bins) is related to the true distribution `x` (n_x bins) by the response matrix `A_hat`:

```
A_hat * x = b
```

where `A_hat_{ij} = P(E_j measured | C_i true)` is the probability that an event generated in true bin j ends up in measured bin i. Direct matrix inversion yields wildly oscillating solutions due to amplification of statistical noise -- the problem is ill-posed.

### Singular Value Decomposition

Any real m x n matrix A can be factored as:

```
A = U * S * V^T
```

where:
- `U` is m x m orthogonal (`U^T * U = I`)
- `V` is n x n orthogonal (`V^T * V = I`)
- `S` is m x n diagonal with non-negative, non-increasing singular values `s_i >= 0`

The singular values encode the "condition" of the matrix. Small singular values correspond to high-frequency (oscillating) components of the solution that are amplified by inversion and cause instability.

### Key Insight: Number-of-Events vs Probability Matrix

A critical recommendation is to use the **number-of-events matrix** A (actual MC counts) rather than the **probability matrix** A_hat (probabilities). This is achieved by normalizing unknowns:

```
w_j = x_j / x_j^ini
```

where `x^ini` is the initial MC distribution. The system becomes:

```
A * w = b,    where A_{ij} = A_hat_{ij} * x_j^ini
```

Two advantages:
1. If `x^ini` is close to truth, `w` is smooth with small bin-to-bin variation, requiring fewer terms in the SVD expansion
2. The number-of-events matrix gives larger weight to well-determined elements (high statistics) and smaller weight to poorly determined elements, automatically accounting for MC statistical uncertainty

### Rescaling Equations

Divide each equation by its error to equalize weights. For covariance matrix B:

```
B = Q * R * Q^T    (SVD of B)
A_tilde_{ij} = (1/r_i) * sum_m Q_{im} * A_{mj}
b_tilde_i = (1/r_i) * sum_m Q_{im} * b_m
```

After this transformation, the covariance of `b_tilde` is the identity matrix I.

### Regularization via Minimum Curvature

Add a regularization term to the least-squares objective:

```
(A_tilde * w - b_tilde)^T * (A_tilde * w - b_tilde) + tau * (C * w)^T * (C * w) = min
```

where C is the second-derivative matrix:

```
C = [ -1  1  0  0  ...
       1 -2  1  0  ...
       0  1 -2  1  ...
       ...         1 -2  1
       ...            1 -1 ]
```

This suppresses solutions with large curvature (rapid oscillation). The regularization parameter `tau` controls the strength.

### SVD Solution with Regularization

Decompose `A_tilde * C^{-1} = U * S * V^T`, then rotate:

```
d = U^T * b_tilde    (rotated r.h.s.)
z = V^T * C * w      (rotated unknowns)
```

The diagonal system `s_i * z_i = d_i` has the regularized solution:

```
z_i^(tau) = d_i * s_i / (s_i^2 + tau)
```

The regularization factor `s_i^2 / (s_i^2 + tau)` acts as a smooth low-pass filter:
- For large `s_i` (low-frequency, well-determined): factor ~ 1 (no suppression)
- For small `s_i` (high-frequency, noise-dominated): factor ~ 0 (suppressed)

The unfolded solution and covariance:

```
w^(tau) = C^{-1} * V * z^(tau)
x_i^(tau) = x_i^ini * w_i^(tau)

Z^(tau)_{ik} = s_i^2 / (s_i^2 + tau)^2 * delta_{ik}
W^(tau) = C^{-1} * V * Z^(tau) * V^T * C^{-T}
X^(tau)_{ik} = x_i^ini * W^(tau)_{ik} * x_k^ini
```

### Inverse Covariance Matrix (Regular, Independent of tau)

A key result: while the covariance matrix `X^(tau)` depends on the regularization parameter, its inverse `X^{-1}` does not:

```
X^{-1}_{jk} = (1 / x_j^ini * x_k^ini) * sum_i A_tilde_{ij} * A_tilde_{ik}
```

This is crucial for chi2 calculations involving the unfolded distribution.

### Choosing the Regularization Parameter tau

Plot `log|d_i|` vs `i`. The effective rank `k` is the index where `|d_i|` drops to the noise level (~1, since the rotated r.h.s. has unit covariance). Then set:

```
tau = s_k^2
```

Components with `i <= k` are statistically significant (signal); components with `i > k` are noise. The distribution of `d_i` for `i > k` should be compatible with standard Gaussian (mean 0, variance 1) -- this also serves as a validation of the error estimates.

### Step-by-Step Algorithm (Section 8)

**Initialization:**
1. Define n_b, n_x, bin boundaries for measured and true distributions
2. Build second-derivative matrix C and compute C^{-1}
3. Generate MC to obtain x^ini and the number-of-events matrix A
4. Read measured distribution b and its covariance matrix B

**Rescaling and rotation:**
1. SVD of covariance B = Q * R * Q^T
2. Rotate/rescale b and A to make covariance = identity: A_tilde, b_tilde
3. Compute inverse covariance X^{-1} (Eq. 53)
4. SVD of A_tilde * C^{-1} = U * S * V^T
5. Compute rotated r.h.s. d = U^T * b_tilde

**Unfolding:**
1. Plot log|d_i| vs i to determine effective rank k
2. Set tau = s_k^2
3. Compute z^(tau), w^(tau), Z^(tau), W^(tau) (Eqs. 48-49)
4. Compute x^(tau) and X^(tau) (Eqs. 51-52)

### 2x2 Example (Pedagogical)

The paper works through a complete 2x2 example with response matrix:

```
A_hat = (1/2) * [ 1+eps  1-eps ]
                 [ 1-eps  1+eps ]
```

where eps in [0,1] controls detector "quality." SVD gives singular values s_1 = 1 and s_2 = eps. For small eps (poor detector), the exact solution amplifies noise by 1/eps. Regularization sets the noisy component to zero, yielding the physically sensible result x_reg = (b_1+b_2)/2 * [1, 1]^T.

## Methodology

The algorithm is entirely linear (loop-free), making it fast and deterministic. The SVD naturally identifies which components of the solution are well-determined (large singular values) and which are noise-dominated (small singular values). The regularization is smooth (no Gibbs phenomenon) and has a single tunable parameter tau whose optimal value can be read directly from the data via the `d_i` spectrum. Full error propagation including the complete covariance matrix and its inverse is built into the algorithm.

## Relevance to PPG12

- **Alternative unfolding method**: SVD unfolding is available in PPG12 via `RooUnfoldSvd` as a cross-check of the default Bayesian method. The interface in ROOT is `TSVDUnfold`
- **Understanding regularization**: The SVD formalism provides intuition for why unfolding is an ill-posed problem and how regularization (whether via early stopping in Bayesian or singular value truncation in SVD) tames noise amplification
- **Response matrix normalization**: The paper's recommendation to use number-of-events matrices (not probability matrices) applies directly to how PPG12 constructs its response matrix from MC
- **Covariance matrix**: The SVD method provides the full covariance matrix and its inverse, important for chi2 comparisons between unfolded data and theory
- **Effective rank determination**: The `|d_i|` spectrum diagnostic could validate PPG12's choice of iteration count in Bayesian unfolding -- both methods are fundamentally controlling the same trade-off between bias and variance
- **pT bins**: PPG12's choice of n_x <= n_b (12 reco bins with extended truth bins) is consistent with the paper's requirement that n_b >= n_x
- **Systematic variation**: Running SVD unfolding alongside Bayesian unfolding (varying the regularization strength) could provide a systematic uncertainty on the unfolding procedure
