# Split Training Pipeline

Trains all 11 photon-ID BDT variants and the NPB score model on the `_split` input files,
in parallel on Condor. Outputs are loaded by `apply_BDT.C` when `use_split_bdt: 1` in
`config_nom.yaml`.

---

## Prerequisites

The following `_split` txt files must exist in the working directory:

| Role | Files |
|------|-------|
| Signal (photon-ID) | `shapes_split_photon5.txt`, `shapes_split_photon10.txt`, `shapes_split_photon20.txt` |
| Background (photon-ID) | `shapes_split_jet5.txt`, `shapes_split_jet12.txt`, `shapes_split_jet20.txt`, `shapes_split_jet30.txt`, `shapes_split_jet40.txt` |
| Signal (NPB) | same jet files as above |
| Background (NPB) | `shapes_data_npb.txt` (real data, no split version needed) |

Generated by `BDTinput.C`. The `logs/` directory must also exist:
```bash
mkdir -p logs
```

---

## Pipeline A: Photon-ID BDT (11 models, Condor)

### Step 1 — Generate per-model configs
```bash
python make_split_configs.py
```
Writes `split_configs/config_{name}_split.yaml` for all 11 models plus `split_configs/filelist.txt`.
Re-run this whenever `make_split_configs.py` is edited.

Spot-check one config:
```bash
head -30 split_configs/config_base_v3E_split.yaml
```

### Step 2 — Submit
```bash
condor_submit submit_split_training.sub
```
Queues 11 jobs (one per YAML), 8 GB RAM / 4 CPUs each. Monitor:
```bash
condor_q
```

### Step 3 — Verify outputs
```bash
ls binned_models/model_*_split_single_tmva.root
```
Expected: `model_base_split_single_tmva.root` through `model_base_v3E_split_single_tmva.root` (11 files).

---

## Pipeline B: NPB Score Model (single job, local or Condor)

### Train
```bash
python train_npb_score.py --config config_npb_training_split.yaml
```
Outputs `npb_models/npb_score_split_tmva.root`.

### Verify
```bash
root -l npb_models/npb_score_split_tmva.root <<< '.ls'
```

---

## Applying the split models

In `config_nom.yaml`, set:
```yaml
use_split_bdt: 1
```
`apply_BDT.C` will then load `model_{name}_split_single_tmva.root` and `npb_score_split_tmva.root`
automatically. Set back to `0` to use the standard (non-split) models.

---

## Troubleshooting

**Check a failed Condor job:**
```bash
cat logs/split_train_<Cluster>.<Process>.err
```

**Re-run a single model locally:**
```bash
python train_split_model.py split_configs/config_base_v3E_split.yaml
```

**Background file has 0 entries** — the txt file exists but is empty or missing. Check that
`BDTinput.C` was run for that jet pT sample.

## File Reference

| File | Description |
|------|-------------|
| `make_split_configs.py` | Generates `split_configs/` YAMLs and `filelist.txt` |
| `train_split_model.py` | CLI wrapper: runs `BinnedTrainingPipeline.run_training()` + `save_results()` |
| `run_split_training.sh` | Condor executable — sources env, runs `train_split_model.py` |
| `submit_split_training.sub` | Condor submission file — queues from `split_configs/filelist.txt` |
| `config_npb_training_split.yaml` | NPB score training config using split jet files |
| `split_configs/` | Generated per-model configs (created by Step A1) |
| `binned_models/` | Photon-ID TMVA ROOT outputs |
| `npb_models/` | NPB score TMVA ROOT outputs |
| `logs/` | Condor stdout/stderr (`split_train_<Cluster>.<Process>.out/err`) |
