\section*{TL;DR}

\begin{itemize}
\item The current ``nominal'' reco-vertex reweight (\texttt{vertex\_reweight\_on: 1}) \textbf{silently drops MC events with $z_{\rm reco}\!=\!-9999$} (no global vertex reconstructed, i.e.\ MBD trigger failed) because they fall outside the reweight-histogram range and get \texttt{vertex\_weight=0}. This \emph{inflates} the MBD$\times$vertex efficiency from the physical $\sim 0.63$ to $\sim 0.89$ (pT-averaged), which in turn \emph{under-estimates} the cross section by $\sim 28$--$40\%$ across the reco fiducial range.
\item If the luminosity is \textbf{raw beam-delivered} (not MBD-triggered), the correct fix is to reweight on the \textbf{truth} vertex $z_h$ instead. Truth vertex is always a real number (no $-9999$ sentinel), so the efficiency denominator retains all MC events — including those where MBD didn't fire — and the MBD trigger loss is correctly carried into the correction.
\item \textbf{Numerical result}: truth-vertex-reweight MBD$\times$vertex efficiency (0.633 pT-avg) $\approx$ no-reweight (0.631) — i.e.\ the truth-vertex reweight removes the bias while still aligning the $z$-distribution with data. Cross section shifts $-23\%$ to $-40\%$ vs.\ the OLD nominal (equivalently, the OLD nominal was $+29\%$ to $+67\%$ too low).
\item \textbf{Recommendation}: switch the nominal configuration from \texttt{vertex\_reweight\_on: 1} to \texttt{truth\_vertex\_reweight\_on: 1} (set \texttt{vertex\_reweight\_on: 0} for clarity even though the truth switch forces it off internally). Treat the reco-vtx-reweight variant as a closure cross-check only.
\end{itemize}

Only the per-event multiplicative weight changes.

\section*{2. The bias in the OLD method}

Decomposition on a photon10 slimtree (previous investigation):
\begin{itemize}
\item 23\% of MC events have $z_{\rm reco}=-9999$ — no global vertex reconstructed because MBD did not fire (the global vertex algorithm uses MBD timing). These events \emph{automatically} fail both $|z_{\rm reco}|<60$ and ${\rm MBD}_{N/S}\ge1$.
\item In the OLD reweight, the reweight histogram covers $[-100,+100]$ cm. $z_{\rm reco}=-9999$ is outside this range, so \texttt{vertex\_weight=0}, meaning these 23\% of events contribute \emph{nothing} to either the numerator or the denominator of the MBD$\times$vertex ratio.
\item The ratio is therefore computed on the 77\% subset where MBD fired, giving an effective ``MBD-conditional" efficiency of $\sim 0.89$, not the physical $\sim 0.63$.
\item The luminosity used in the nominal config (\texttt{lumi: 16.2735}\,pb$^{-1}$, \texttt{config\_bdt\_nom.yaml:246}) is raw beam-delivered (sourced from \texttt{lumi/60cmLumi\_fromJoey.list}, scaled by the $|z|<60$ fiducial factor but \emph{not} by MBD trigger efficiency), so the efficiency denominator must include the MBD trigger loss. OLD underestimates the correction, so $\sigma = N_{\rm data}/(\varepsilon\cdot L)$ comes out \textbf{too low} by $\sim (1-0.77)\approx 23\%$ at low pT and up to $\sim 40\%$ at high pT.
\end{itemize}

The truth-vertex reweight fixes this because $z_h$ is the generator-level vertex — always a finite number, never a sentinel. Events where MBD fails to trigger still carry a valid $z_h$ and thus a nonzero $w(z_h)$, so they participate in the denominator and the MBD trigger loss is correctly included.

\subsection*{3.3 Why TRUTH sits \emph{between} NO and OLD in cross section}

MBD$\times$vertex efficiency is almost identical for NO and TRUTH (Tab.~\ref{tab:mbdeff}), yet the cross sections differ by 6--15\%. The residual lives in the \emph{other} efficiencies folded into \texttt{total\_eff} and the unfolding response matrix:
\[
\sigma = \frac{N_{\rm data}^{\rm A}}{\varepsilon_{\rm reco}\,\varepsilon_{\rm iso}\,\varepsilon_{\rm id}\,\varepsilon_{\rm MBD\times vtx}\,L},
\qquad\varepsilon_{\rm MBD\times vtx}({\rm NO}) \approx \varepsilon_{\rm MBD\times vtx}({\rm TRUTH}).
\]
The reco/iso/ID efficiencies and the unfolding response are computed from MC events that pass $|z_{\rm reco}|<60$. Reweighting to the data $z_h$ distribution (narrower than MC's native flat $\pm 100$\,cm) shifts which MC events dominate these averages, modestly raising the average reco efficiency (less $\eta$-acceptance loss at small $|z|$). TRUTH therefore has slightly smaller $\sigma$ than NO — exactly the first-order benefit one expects from aligning MC to data.

\section*{4. Physics checks}

\begin{itemize}
\item \textbf{Is the luminosity raw beam-delivered?} Confirmed (Shuonli): the \texttt{lumi: 16.2735}\,pb$^{-1}$ in \texttt{config\_bdt\_nom.yaml} is from \texttt{lumi/60cmLumi\_fromJoey.list}, which is beam-delivered lumi scaled by the $|z|<60$ fiducial factor. It is \emph{not} MBD-triggered, so the efficiency denominator must include MBD trigger loss.
\item \textbf{Does TRUTH reweight truly avoid the $-9999$ filter?} Verified at \texttt{RecoEffCalculator\_TTreeReader.C:1429-1434}: \texttt{vertex\_weight = h\_w\_iterative->Interpolate(vertexz\_truth)} with \texttt{vertexz\_truth} read from the MC truth record, which is always a finite number. Events where MBD failed to fire have $z_{\rm reco}\!=\!-9999$ but $z_h\neq-9999$, so they get a nonzero weight.
\item \textbf{Closure between NO and TRUTH on $\varepsilon_{\rm MBD\times vtx}$}: NO$-$TRUTH $\lesssim 0.5\%$ per pT bin, mostly inside statistical error. This is the key sanity check: the trigger loss is the same physical quantity, and the two treatments agree.
\item \textbf{MC statistical uncertainty}: bin-to-bin ratios are smooth (no pT spikes), consistent with the ratios being systematic rather than statistical.
\item \textbf{Closure caveat on the truth reweight file}: \texttt{truth\_vertex\_reweight/output/1p5mrad/reweight.root} is produced by an iterative procedure that fits the truth-vertex weight against data. It is \emph{not} fully decoupled from the data it is applied to, so there is a residual circularity concern — the reweight could in principle absorb genuine MC--data shape mismatches from other sources (e.g.\ missing MC samples, detector miscalibration). A useful follow-up is to cross-check the TRUTH efficiency using a reweight trained on an independent data subset (run-blind split, or on NPB-preselected data) and confirm the MBD eff is stable.
\end{itemize}

\section*{5. Recommendations (actionable)}

\begin{enumerate}
\item \textbf{Change the nominal 1.5-mrad config} \texttt{efficiencytool/config\_bdt\_nom.yaml} to set \texttt{vertex\_reweight\_on: 0} and \texttt{truth\_vertex\_reweight\_on: 1}. (The truth switch internally forces \texttt{vertex\_reweight\_on=0}, but setting both explicitly is clearer for reviewers.) Re-run \texttt{oneforall\_tree\_and\_yield.sh config\_bdt\_nom.yaml}; cross section will shift up by $\sim$28--67\% per pT bin (Table~\ref{tab:xs}).
\item \textbf{Do the same for the 0-mrad and all-runs nominal configs} (and use the 0-mrad \texttt{truth\_vertex\_reweight\_file}). The 22.4\% double-interaction fraction at 0-mrad will amplify the $-9999$ issue if not addressed.
\item \textbf{Re-run all BDT systematic variations} (\texttt{make\_bdt\_variations.py}) from the new truth-vtx nominal. The 30--70\% shift in absolute cross section will propagate into any ratio-based systematic that is currently evaluated relative to the OLD nominal; the \emph{relative} fractional uncertainties should be nearly invariant but the absolute bands change.
\item \textbf{Demote the reco-vtx reweight variant} (\texttt{config\_bdt\_vtxreweight0.yaml}) in \texttt{make\_bdt\_variations.py}: change \texttt{syst\_role} from \texttt{one\_sided} to a closure-test label, or remove from the systematic envelope. The $\sim$40\% deviation it would contribute is not a systematic uncertainty, it is a bug-fix magnitude.
\item \textbf{Update the wiki} \texttt{wiki/pipeline/04-efficiency-yield.md} to document:
  \begin{enumerate}
    \item The $z_{\rm reco}=-9999$ sentinel and its interaction with the reco-vtx reweight.
    \item The truth-vtx reweight as the correct nominal for raw beam lumi.
    \item The dead code \texttt{mbdcorr = 25.2/42/0.57} at \texttt{CalculatePhotonYield.C:54}.
  \end{enumerate}
\item \textbf{Follow-up (optional)}: verify 0-mrad and all-runs truth-vtx reweight files exist (\texttt{efficiencytool/truth\_vertex\_reweight/output/\{0mrad,all\}/reweight.root}). If missing, regenerate with the standalone iterative tool in \texttt{efficiencytool/truth\_vertex\_reweight/}.
\end{enumerate}
